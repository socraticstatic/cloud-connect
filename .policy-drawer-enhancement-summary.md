# Policy Drawer Enhancement Summary

## Overview
Upgraded the routing policy maker in Connection Details from a modal to a comprehensive side drawer with resource selection, traffic direction controls, and action-specific options inherited from Configure > Policies.

## Changes Implemented

### 1. Replaced Modal with Side Drawer

**Before:** Modal (fixed size, limited space)
**After:** Side Drawer (xl size, scrollable, better UX)

#### Benefits
- More space for comprehensive form
- Better organization of sections
- Slides in from right (industry standard)
- Sticky footer with action buttons
- Escape key support
- Click outside to close

### 2. Added Specific Resource Selection

When user selects "Apply to Specific Links/CloudRouters/VNFs", they can now:

#### Links Selection
- Shows all available links in the connection
- Checkbox for each link
- Displays link details:
  - Link name
  - VLAN ID
  - Status (Active/Inactive)
  - IP subnet (if available)
- Counter: "X of Y links selected"
- Max height with scroll for many links

#### Cloud Routers Selection
- Shows all cloud routers
- Displays:
  - Router name
  - Provider (AWS, Azure, GCP, etc.)
  - BGP ASN
- Counter and scroll support

#### VNFs Selection
- Shows all VNFs
- Displays:
  - VNF name
  - Type (Firewall, Load Balancer, etc.)
  - Provider
- Counter and scroll support

### 3. Traffic Direction Control

Added direction selection (inherited from Configure > Policies):

**Options:**
- **On Premise → Partner**: Policy applies to outbound traffic
- **Partner → On Premise**: Policy applies to inbound traffic
- **Both Directions**: Policy applies bidirectionally (default)

This matches the directional toggles in Configure > Policies tabs.

### 4. Action-Specific Options

Options dynamically change based on selected action:

#### Allow Action Options
- ✓ Matching Routes
- ✓ Community Value Filter

#### Deny Action Options
- ✓ Matching Routes
- ✓ Block Default Routes
- ✓ Community Value Filter

#### Manipulate Action Options
- ✓ Prepend Advertisements with Extra BGP ASNs
- ✓ Selective CV Tagging to Routes/Prefixes

#### Advertise Action Options
- ✓ Advertise Static Routes

These options are directly pulled from Configure > Policies structure:
- **Deny Actions** section → Deny options
- **Allow Actions** section → Allow options
- **Manipulations** section → Manipulate options
- **Advanced** section → Advertise options

### 5. Enhanced Form Structure

Organized into logical sections:

#### Section 1: Basic Information
- Policy Name (required)
- Description (optional)
- Priority (1-1000, default: 100)

#### Section 2: Action Configuration
- Action dropdown (Allow/Deny/Manipulate/Advertise)
- Protocol selection (Any/TCP/UDP/ICMP/BGP/OSPF)
- Traffic Direction radio buttons
- Action-specific options (checkboxes)

#### Section 3: Apply To Resources
- Target resource type dropdown
- Conditional resource selection (when not "all")
- Visual list with checkboxes
- Selection counter

#### Section 4: Policy Conditions
- Source Prefix (CIDR notation)
- Destination Prefix (CIDR notation)
- BGP Community (e.g., 65000:100)
- AS Path (regex pattern)

## User Workflow

### Creating a Policy

1. **Click "Add Policy" button**
   - Side drawer slides in from right

2. **Fill Basic Information**
   - Enter policy name
   - Add description (optional)
   - Set priority level

3. **Configure Action**
   - Select action type (Allow/Deny/Manipulate/Advertise)
   - Choose protocol
   - Select traffic direction
   - Check relevant action-specific options

4. **Select Resources**
   - Choose "All Resources" or specific type
   - If specific type, select from checkbox list
   - See selection count update

5. **Define Conditions (Optional)**
   - Enter source/destination prefixes
   - Add BGP community values
   - Specify AS path patterns

6. **Create Policy**
   - Click "Create Policy" button
   - Drawer closes, policy added to list
