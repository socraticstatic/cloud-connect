## 💡 Educational Impact

### **What Users Learn:**

1. **Roles = Permissions**
   - User role → view only
   - Admin role → full CRUD + manage users/billing
   - Super admin → platform-wide access

2. **Filters = Resource Visibility**
   - `owned-by-me` → Only my stuff
   - `my-department` → Department resources
   - `my-pools` → Pool-specific
   - `my-tenant` → Entire tenant
   - `all-tenants` → Cross-tenant

3. **Scope = Location in Hierarchy**
   - Platform → Tenant → Department → Pool → Resource
   - Permissions inherit downward
   - Assign at narrowest scope possible (least privilege)

4. **Putting It All Together**
   ```
   Role: admin (WHAT: view, create, edit, delete)
   Scope: /tenants/acme/depts/engineering (WHERE)
   Filter: my-pools (WHICH: only assigned pools)

   Result: Can perform admin actions on engineering
           resources, but only those in assigned pools
   ```
