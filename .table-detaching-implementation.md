# Table Detaching Implementation - VNF Table

## Overview
Successfully implemented table detaching functionality for the VNF Table as a proof-of-concept. This feature allows users to open the VNF table in a separate browser window for improved multitasking and screen real estate management.

## Implementation Summary

### 1. Persisted Window State (localStorage)
Window state is persisted to the browser's localStorage across sessions:
- Tracks window metadata (position, size, status)
- Scoped to the current browser profile
- Automatic cleanup of old closed windows (7 days)
- Supports up to 15 concurrent detached windows

**Key Features:**
- No backend - everything lives in the browser
- Automatic timestamp management
- Lightweight lookups by window id

#### useVNFSync Hook
**Location:** `src/hooks/useVNFSync.ts`

A custom hook that manages cross-window data synchronization:
- Broadcasts VNF and CloudRouter updates to detached windows
- Receives updates from detached windows
- Handles sync requests from newly opened windows
- Cleans up on parent window close

**Communication Protocol:**
- `REQUEST_SYNC` - Detached window requests initial data
- `FULL_SYNC` - Parent sends complete data set
- `VNF_UPDATED` - Parent broadcasts VNF changes
- `CLOUD_ROUTERS_UPDATED` - Parent broadcasts router changes
- `VNF_UPDATED_FROM_DETACHED` - Detached window made changes
- `DETACHED_CLOSED` - Detached window closed
- `PARENT_CLOSED` - Parent window closing notification

## Technical Architecture

### Data Flow

```
Parent Window (ConnectionDetails)
    ↓
useVNFSync Hook
    ↓
BroadcastChannel('vnf-sync')
    ↓
DetachedVNFTable Component
    ↓
VNFTable Component
```

### Communication Pattern

1. **Initial Load:**
   - Detached window opens and sends `REQUEST_SYNC`
   - Parent responds with `FULL_SYNC` containing current data
   - Detached window renders with synced data

2. **Parent Updates:**
   - User makes changes in parent window
   - useVNFSync broadcasts `VNF_UPDATED` or `CLOUD_ROUTERS_UPDATED`
   - Detached window receives and applies updates

3. **Detached Updates:**
   - User makes changes in detached window
   - Detached window broadcasts `VNF_UPDATED_FROM_DETACHED`
   - Parent window receives and applies updates
   - Parent broadcasts to other detached windows

4. **Cleanup:**
   - Window closing broadcasts `DETACHED_CLOSED` or `PARENT_CLOSED`
   - Other windows handle cleanup and state updates
