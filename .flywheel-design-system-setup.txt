DESIGN PRINCIPLES
-----------------

1. SEMANTIC NAMING
   - Use meaning-based tokens (heading, body, link)
   - Not presentation-based (blue, red, large)
   - Easier to maintain and understand

2. DARK MODE READY
   - All tokens have dark mode variants
   - Automatic switching via CSS
   - Proper contrast maintained

3. CONSISTENCY
   - Single source of truth (tokens.css)
   - Same colors everywhere
   - Easier updates and changes

4. ACCESSIBILITY
   - WCAG 2.1 AA compliant contrast ratios
   - Text/background combinations tested
   - Focus states properly defined

5. BACKWARD COMPATIBILITY
   - Legacy colors still available
   - Gradual migration path
   - No breaking changes

TOKEN NAMING CONVENTION
-----------------------
Format: {prefix}-{semantic}-{variant}

Prefixes:
• fw = Flywheel namespace
• text = Text colors
• bg = Background colors
• border = Border colors
• ring = Focus ring colors

Semantic Names:
• heading, body, bodyLight (text hierarchy)
• link, linkHover, linkPrimary (interactions)
• base, wash, neutral (backgrounds)
• primary, secondary, active (borders)
• success, warn, error, info (status)

Variants:
• 100-900 (intensity scale)
• 000 (special lighter variant)
• light, functional (named variants)
