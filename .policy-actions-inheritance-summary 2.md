# Policy Actions Inheritance Implementation Summary

## Overview
Updated the Connection Details Policies tab to inherit the same policy action choices (Allow, Deny, Manipulate, Advertise) that are used in the Configure > Policies section, creating consistency across the application.

## Problem Identified
The routing policy rule maker in Connection Details was using generic actions (permit/deny) instead of the specific, business-aligned actions defined in Configure > Policies (Allow, Deny, Manipulate, Advertise).

## Solution Implemented

### 1. Updated Type Definitions

**File**: `src/types/routingPolicy.ts`

**Before:**
```typescript
export type PolicyAction = 'permit' | 'deny';
```

**After:**
```typescript
export type PolicyAction = 'allow' | 'deny' | 'manipulate' | 'advertise';
```

This aligns with the Configure > Policies structure:
- **Allow**: Matching routes, community value filters (from Allow Actions section)
- **Deny**: Block routes, filters (from Deny Actions section)
- **Manipulate**: AS path prepending, CV tagging (from Manipulations section)
- **Advertise**: Static route advertisements (from Advanced section)

### 2. Updated Sample Policies

**File**: `src/components/connection/tabs/PoliciesTab.tsx`

Created four sample policies demonstrating each action type:

1. **Production Traffic Priority (Allow)**
   - Priority: 100
   - Protocol: Any
   - Applies to: Links
   - Purpose: Allow production traffic matching prefix/community

2. **Block Suspicious Traffic (Deny)**
   - Priority: 200
   - Protocol: TCP
   - Applies to: All resources
   - Purpose: Deny traffic from malicious sources

3. **BGP AS Path Prepending (Manipulate)**
   - Priority: 75
   - Protocol: BGP
   - Applies to: Cloud Routers
   - Purpose: Manipulate route advertisements via AS path prepending

4. **Advertise Static Routes (Advertise)**
   - Priority: 50
   - Protocol: BGP
   - Applies to: VNFs
   - Purpose: Advertise static routes to BGP neighbors

### 3. Updated Action Filters

**Filter Options Updated:**
- Before: `['permit', 'deny']`
- After: `['allow', 'deny', 'manipulate', 'advertise']`

All four action types now available in the filter dropdown for policy search and filtering.

### 4. Updated Action Badges

Added color-coded visual indicators for each action type:

| Action | Color | Icon | Use Case |
|--------|-------|------|----------|
| **Allow** | Green (`status-active`) | Check ✓ | Permitting traffic |
| **Deny** | Red (`status-error`) | Alert Triangle ⚠ | Blocking traffic |
| **Manipulate** | Orange (`status-warning`) | Edit ✏ | Modifying route attributes |
| **Advertise** | Blue (`fw-accent`) | Play ▶ | Route advertisements |

### 5. Updated Modal Form

**Add/Edit Policy Modal Changes:**

**Before:**
```tsx
<FormField label="Action" required>
  <select>
    <option value="permit">Permit</option>
    <option value="deny">Deny</option>
  </select>
</FormField>
```

**After:**
```tsx
<FormField label="Action" required helpText="Inherited from Configure > Policies">
  <select>
    <option value="allow">Allow</option>
    <option value="deny">Deny</option>
    <option value="manipulate">Manipulate</option>
    <option value="advertise">Advertise</option>
  </select>
</FormField>
```

Added help text to clarify that these actions are inherited from the Configure > Policies section.
