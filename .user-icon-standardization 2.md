# User Icon Standardization - Flywheel Compliant

## 🎯 **Problem Solved**

**Before:** User icons had inconsistent sizing and colors across the site
- Mix of `h-3`, `h-4`, `h-5`, `h-8`, `h-12` classes
- Inconsistent colors: `text-gray-400`, `text-gray-500`, `text-blue-600`, `text-fw-link`, etc.
- Direct `<User />` imports scattered throughout codebase
- No single source of truth for styling

**After:** One standard, minimal, Flywheel-compliant UserIcon component
- Consistent sizing with semantic names
- Flywheel color tokens
- Single import path
- Easy to maintain

---

## ✨ **UserIcon Component**

### **Location**
`src/components/common/UserIcon.tsx`

### **API**

```typescript
interface UserIconProps {
  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl';
  variant?: 'default' | 'primary' | 'success' | 'muted' | 'inherit';
  className?: string;
}
```

### **Size Standards (Flywheel-aligned)**

| Size | Tailwind | Pixels | Use Case |
|------|----------|--------|----------|
| `xs` | `h-3 w-3` | 12px | Micro indicators, badges |
| `sm` | `h-4 w-4` | 16px | Inline text icons, table cells |
| `md` | `h-5 w-5` | 20px | Standard icons (DEFAULT) |
| `lg` | `h-8 w-8` | 32px | Feature icons, cards |
| `xl` | `h-12 w-12` | 48px | Large display, empty states |

### **Color Variants (Flywheel tokens)**

| Variant | Token | Color | Use Case |
|---------|-------|-------|----------|
| `default` | `text-fw-body` | Gray-600 | Secondary elements |
| `primary` | `text-fw-link` | Functional Blue | Interactive, emphasis |
| `success` | `text-fw-success` | Green | Active states, success |
| `muted` | `text-fw-disabled` | Gray-400 | Disabled, inactive |
| `inherit` | (none) | Inherited | Custom contexts |

## 🔍 **Remaining User Icons**

### **Files Still Using Direct `<User />` Import:**

Some files still use `<User />` for specific contexts (non-avatar usage):
- Navigation context icons
- Settings/profile icons
- Mixed icon sets in feature cards

These are intentionally left as-is where `UserIcon` doesn't apply (e.g., not representing user avatars).

### **Migration Strategy**

**When to use UserIcon:**
- ✅ User avatars
- ✅ User profile indicators
- ✅ Member lists
- ✅ User-related metrics
- ✅ Role indicators (for "user" role)

**When to keep `<User />`:**
- ❌ Navigation menu items
- ❌ Mixed icon sets in cards
- ❌ Non-avatar user representations
- ❌ Settings/preferences icons
