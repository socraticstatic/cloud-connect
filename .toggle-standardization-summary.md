# Toggle Component Standardization Summary

## Problem Identified

The routing policy toggle switches looked different from the toggle in the Connection Wizard because different toggle implementations were being used across the site. This created visual inconsistency in the UI.

## Solution Implemented

Created a reusable `Toggle` component with consistent styling that matches the design from the Connection Wizard and applied it across all toggle switches in the application.

## Changes Made

### 1. Created New Toggle Component

**File**: `src/components/common/Toggle.tsx`

A reusable, accessible toggle switch component with:

- **Consistent Styling**: Matches the w-10 h-5 dimensions from the wizard
- **AT&T Branding**: Uses `bg-[#003184]` (AT&T Blue) for checked state
- **Flywheel Tokens**: Uses `bg-fw-disabled` for unchecked state
- **Accessibility**: Includes sr-only checkbox and proper labels
- **Size Options**: Supports 'sm' and 'md' sizes
- **Disabled State**: Visual indication with opacity
- **Smooth Animation**: 200ms ease-in-out transitions
- **Memoized**: Performance optimized with React.memo

#### Design Specifications
- **Checked**: AT&T Blue (#003184) background
- **Unchecked**: Gray (flywheel disabled token) background
- **Toggle Dot**: White with shadow-md
- **Dimensions**: w-10 h-5 (medium), w-9 h-4 (small)
- **Animation**: Translates 5 units when checked
- **Border Radius**: Fully rounded (rounded-full)

### 2. Updated Components to Use New Toggle

#### PoliciesTab (`src/components/connection/tabs/PoliciesTab.tsx`)
- **Before**: Custom button with `h-6 w-11` and manual state classes
- **After**: Clean `<Toggle>` component
- **Benefit**: Consistent styling, simpler code

#### ConnectionWizard (`src/components/wizard/ConnectionWizard.tsx`)
- **Before**: Inline toggle with manual HTML structure

- **Benefit**: Reduced code duplication, easier maintenance

#### AgenticAISettings (`src/components/configure/AgenticAISettings.tsx`)

- **Benefit**: Much cleaner JSX, consistent appearance

### 3. Component API

```tsx
interface ToggleProps {
  checked: boolean;              // Current toggle state
  onChange: (checked: boolean) => void;  // Callback with new state
  label?: string;                // Optional text label
  disabled?: boolean;            // Disable interaction
  size?: 'sm' | 'md';           // Size variant
  className?: string;            // Additional classes
}
```

#### Usage Examples

**Basic Toggle:**
```tsx
<Toggle
  checked={isEnabled}
  onChange={setIsEnabled}
/>
```

**Toggle with Label:**
```tsx
<Toggle
  checked={showAI}
  onChange={setShowAI}
  label="AI Assistant"
/>
```

**Disabled Toggle:**
```tsx
<Toggle
  checked={isLocked}
  onChange={setIsLocked}
  disabled={!hasPermission}
  label="Feature Lock"
/>
```

**Small Size:**
```tsx
<Toggle
  checked={miniMode}
  onChange={setMiniMode}
  size="sm"
/>
```

## Migration Guide for Developers

If you find an old-style toggle in the codebase:

**Old Pattern:**
```tsx
<label className="relative inline-flex items-center cursor-pointer">
  <input
    type="checkbox"
    checked={value}
    onChange={(e) => setValue(e.target.checked)}
    className="sr-only peer"
  />
  <div className="w-11 h-6 bg-gray-200 peer-checked:bg-blue-600 rounded-full...">
    <div className="...translate-x-..."></div>
  </div>
</label>
```

**New Pattern:**
```tsx
import { Toggle } from '../common/Toggle';

<Toggle
  checked={value}
  onChange={setValue}
/>
```
