# LMCC GA Phase 4 Implementation Plan — Commercial Flows

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bandwidth change as an asymmetric commercial flow (upgrade free / downgrade fee + acknowledgment / migration failure as a real state) and delete as a commercial event with the early-termination charge surfaced before confirm.

## Global Constraints

- Up and down are NEVER one neutral control. Downgrade fee + new rate shown BEFORE confirm, explicit acknowledgment required.
- The portal displays fees from the (mock) system of record; it never claims to calculate them.
- Delete framing: "deleting ends your service and may trigger a charge" — never merely "billing stops".
- Deleting → Deleted are visible states; recurring billing ends at delete.
- "Could not complete provisioning seamlessly" (Interconnect Migration) is a real, reachable state → maps to Needs attention.

---

### T1: Commercial math (pure, tested) — `src/utils/lmccBilling.ts`
- `changeDirection(currentMbps, newMbps): 'upgrade' | 'downgrade' | 'none'`
- `downgradeChangeFee(currentMbps, newMbps): number` — mock: $500 + 10% of the monthly-rate delta.
- `earlyTerminationCharge(mbps, monthsRemaining): number` — mock: 50% of remaining commitment.
- Tests for direction, fee monotonicity, zero-remaining termination = 0.

### T2: BandwidthChangeModal + panel hookup
- `src/components/connection/lmcc/BandwidthChangeModal.tsx`: current vs new (GA tiers), live rate delta; upgrade shows "No penalty · no outage"; downgrade shows change fee (system-of-record framing) + required ack checkbox; 100 Gbps adds an interconnect-migration notice + a demo "simulate migration failure" affordance; submit → working state (~1.5s) → success updates `connection.bandwidth`, logs `bandwidth-changed`, toast; failure sets the connection to Needs attention (status 'Inactive' displays as such for LMCC) + logs.
- LMCCStatusPanel: bandwidth control becomes read value + "Change bandwidth" button opening the modal (type select unchanged).

### T3: Delete as commercial event
- `types/connection.ts`: add 'Deleting' to ConnectionStatus.
- connectionSlice: `deleteLmccConnection(id)` — allowed on Live; sets Deleting, then Deleted after a beat; logs `deleted` ("Service ended — early-termination charge may apply; recurring billing stopped."). Non-LMCC `removeConnection` untouched.
- ConnectionDetails delete confirm for LMCC: plain consequence statement, the termination charge amount (from T1, framed as from the billing system of record), months remaining, explicit checkbox, confirm label "Delete and end service".

### T4: E2E + memory
- Browser: downgrade shows fee + requires ack; upgrade shows no-penalty; migration failure reachable → Needs attention explainer appears; delete flow shows charge → Deleting → Deleted; activity log carries both events. Full suite baseline. Memory.
