# Impeccable — NetBond Advanced

## Design Context

### Users
Enterprise network engineers and AT&T stakeholders. Primary consumption context: live stakeholder demos of a cloud-connectivity management portal (hubs, connections, VNFs, network topology). This is a high-fidelity stakeholder mock — no backend; optimize for how it looks and demos.

### Brand Personality
Calm, dense, authoritative. AT&T enterprise: blue-forward, restrained, information-first. Confidence over delight; nothing playful, nothing decorative.

### Aesthetic Direction
Light theme product UI. Flat, quiet chrome; data is the hero. Anti-references: dashboard-template look, pill-everything, boxed tables, gradient accents.

### Design Principles (settled — do not revert)
1. Tables: no box wrapper, no horizontal scroll; `table-fixed` + truncate to fit. Not everything is a pill — resiliency is plain text, status is dot + text.
2. Selected horizontal tabs use a border-b-2 underline, never a rounded pillbox.
3. Hubs and Connections tabs share ConnectionGrid; both must honor mini/full cards, list, topology view modes symmetrically.
4. The 'conn-card' Fields gear drives both connection cards and per-type list tables — one control.
5. Detail pages (Connection & Hub) share the full 10-tab set, all always active; VNF labeled "VNF/VNFs"; hub connection lists group by type.
6. Create wizard is hub-first, end-to-end, with the read-only "You're building" MiniTopology strip.
7. "Cloud Router" is now "Gateway" — never revert.
