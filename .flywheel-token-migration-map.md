# Flywheel Design Token Migration Map

This document provides a complete mapping from legacy tokens to AT&T Flywheel Design System tokens.

## Background Colors

| Legacy Token | Flywheel Token | Use Case |
|--------------|----------------|----------|
| `bg-white` | `bg-fw-base` | Primary backgrounds |
| `bg-gray-50` | `bg-fw-wash` | Light wash backgrounds |
| `bg-gray-100` | `bg-fw-wash` | Subtle backgrounds |
| `bg-gray-200` | `bg-fw-neutral` | Neutral backgrounds |
| `bg-blue-50` | `bg-fw-accent` | Accent/highlight backgrounds |
| `bg-brand-lightBlue` | `bg-fw-accent` | Brand accent backgrounds |

## Text Colors

| Legacy Token | Flywheel Token | Use Case |
|--------------|----------------|----------|
| `text-gray-900` | `text-fw-heading` | Headings, primary emphasis |
| `text-gray-800` | `text-fw-heading` | Strong text |
| `text-gray-700` | `text-fw-body` | Body text |
| `text-gray-600` | `text-fw-bodyLight` | Secondary/light body text |
| `text-gray-500` | `text-fw-disabled` | Muted/disabled text |
| `text-gray-400` | `text-fw-disabled` | Disabled states |
| `text-brand-blue` | `text-fw-link` | Links, interactive text |
| `text-blue-600` | `text-fw-link` | Links |

## Border Colors

| Legacy Token | Flywheel Token | Use Case |
|--------------|----------------|----------|
| `border-gray-200` | `border-fw-secondary` | Subtle borders, dividers |
| `border-gray-300` | `border-fw-secondary` | Default borders |
| `border-gray-600` | `border-fw-primary` | Strong borders |
| `border-gray-800` | `border-fw-focus` | Focus states |
| `border-brand-blue` | `border-fw-active` | Active/selected borders |
| `border-blue-600` | `border-fw-active` | Interactive borders |

## Interactive States

| Legacy Token | Flywheel Token | Use Case |
|--------------|----------------|----------|
| `brand-blue` | `fw-blue-functional` or `fw-link` | Primary interactive color |
| `hover:bg-gray-50` | `hover:bg-fw-wash` | Subtle hover backgrounds |
| `hover:bg-gray-100` | `hover:bg-fw-neutral` | Hover backgrounds |
| `hover:text-gray-700` | `hover:text-fw-body` | Hover text |
| `focus:ring-brand-blue` | `focus:ring-fw-active` | Focus rings |
| `focus:border-brand-blue` | `focus:border-fw-active` | Focus borders |

| `bg-red-50` | `bg-[var(--status-error-bg)]` | Error backgrounds |
| `text-red-600` | `text-fw-error` | Error text |

| `text-orange-600` | `text-fw-warn` | Warning text |
| `border-orange-600` | `border-fw-warn` | Warning borders |

## Component-Specific Mappings

### Cards
```tsx
// Legacy
<div className="bg-white rounded-lg border border-gray-200">

// Flywheel
<div className="bg-fw-base rounded-lg border border-fw-secondary">
```

### Buttons (Primary CTA)
```tsx
// Legacy
<button className="bg-brand-blue text-white hover:bg-blue-700">

// Flywheel
<button className="bg-fw-ctaPrimary text-fw-linkPrimary hover:bg-fw-ctaPrimaryHover">
```

### Form Inputs
```tsx
// Legacy
<input className="border-gray-300 focus:border-brand-blue focus:ring-brand-blue" />

// Flywheel
<input className="border-fw-secondary focus:border-fw-active focus:ring-fw-active" />
```

### Navigation Active States
```tsx
// Legacy
<a className="text-brand-blue border-b-2 border-brand-blue">

// Flywheel
<a className="text-fw-link border-b-2 border-fw-active">
```

### Tables
```tsx
// Legacy
<thead className="bg-gray-50">
  <th className="text-gray-500">

// Flywheel
<thead className="bg-[var(--table-header-bg)]">
  <th className="text-[var(--table-header-text)]">
```

## Search & Replace Patterns

### Safe Global Replacements
These can be done with find/replace across all files:

1. `bg-white` → `bg-fw-base` (in component classNames)
2. `border-gray-200` → `border-fw-secondary`
3. `text-gray-900` → `text-fw-heading`
4. `text-gray-700` → `text-fw-body`
5. `text-gray-600` → `text-fw-bodyLight`

### Context-Sensitive Replacements
These require reviewing each instance:

1. `text-gray-500` → Could be `text-fw-disabled` or `text-fw-bodyLight` depending on context
2. `bg-gray-50` → Could be `bg-fw-wash` or component-specific token
3. Brand blue variations → Need to determine if link, active, or CTA

## Notes

- The `.card` CSS class already uses generic styles, but instances using direct Tailwind classes need migration
- Some components use inline hex colors that should be replaced with semantic tokens
- Pay special attention to status indicators and badges for consistent semantic color usage
- Navigation components have special border-radius requirements (no-rounded class) - preserve these
