## 🎯 What Was Wrong

### The Fundamental Misunderstanding

We were using "scope" to mean **resource filtering** (which resources to show), when scope actually means **hierarchical location** (where in the resource tree permissions apply).

**Incorrect Usage:**
```typescript
// ❌ WRONG - This is actually a resource filter, not a scope
scope: 'own' | 'department' | 'pool' | 'tenant' | 'platform'
```

**Correct Usage:**
```typescript
// ✅ CORRECT - Scope is a hierarchical path
scope: '/tenants/acme-corp/departments/engineering'

// ✅ CORRECT - Separate concept for filtering
resourceFilter: 'owned-by-me' | 'my-department' | 'my-pools' | 'my-tenant' | 'all-tenants'
```

## 🔑 Key Concepts (Corrected)

### What Each Term Means

| Term | Definition | Example |
|------|------------|---------|
| **Scope** | WHERE in resource hierarchy | `/tenants/acme-corp/departments/engineering` |
| **Role** | WHAT permissions | `admin` (has view, create, edit, delete, manage_users) |
| **Resource Filter** | WHICH resources within scope | `my-pools` (only pools I'm assigned to) |
| **Permission** | Specific action | `view`, `create`, `edit`, `delete` |
| **Role Assignment** | Principal + Role + Scope | User-123 has admin at /tenants/acme |

### The Formula

```
Role Assignment = WHO + WHAT + WHERE + (optionally) WHICH

Example:
- WHO: user-123
- WHAT: admin role (view, create, edit, delete, manage_users)
- WHERE: /tenants/acme-corp/departments/engineering
- WHICH: my-pools filter

Result: User-123 can perform admin actions on engineering department
        resources, but only those in pools they're assigned to.
```

## 📊 Before vs After

### Permission Check Example

**Scenario:** Can user edit connection-123 in engineering department?

**Before (INCORRECT):**
```typescript
// Check if user's role has permission
const hasPermission = ROLE_PERMISSIONS[userRole].includes('edit');

// Then check "scope" (actually a filter)
if (userScope === 'own' && resource.ownerId !== userId) {
  return false;
}
```

**After (CORRECT):**
```typescript
// Check if any role assignment covers the resource's scope
const assignments = getUserAssignments(userId);
const applicable = assignments.filter(a =>
  ScopePathBuilder.isParentOf(
    a.scope,
    '/tenants/acme-corp/departments/engineering/connections/connection-123'
  )
);

// Check permissions and filters
for (const assignment of applicable) {
  if (assignment.role.permissions.includes('edit')) {
    if (passesResourceFilter(assignment, resource)) {
      return true; // ✅ Allowed
    }
  }
}
```
