# User Avatar Standardization - Flywheel Minimal Design

## 🎯 **Problem Identified**

User avatars were **inconsistent across the site** with multiple visual styles:

### **Issues Found:**
1. ❌ Gradient backgrounds: `bg-gradient-to-br from-fw-blue-000 to-fw-accent`
2. ❌ Multiple solid colors: `bg-blue-500`, `bg-blue-100`, `bg-gray-200`
3. ❌ Different border styles: `border-2 border-fw-secondary`, `border-2 border-white`, `border border-gray-200`
4. ❌ Inconsistent icon colors: Some white, some primary, some muted

**Result:** Different pages showed different avatar styles (Users page vs Activity page vs Groups page)

---

## ✨ **Solution: One Standard Avatar Style**

### **Flywheel-Compliant Avatar Container:**

```tsx
<div className="h-10 w-10 rounded-full bg-fw-blue-light flex items-center justify-center border border-fw-secondary">
  <UserIcon size="md" variant="primary" />
</div>
```

### **Design Specifications:**

**Background:**
- ✅ `bg-fw-blue-light` - Light blue wash (Flywheel token)
- ❌ No gradients
- ❌ No solid blue (`bg-blue-500`)
- ❌ No gray (`bg-gray-200`)

**Border:**
- ✅ `border border-fw-secondary` - Single pixel, Flywheel token
- ❌ Not `border-2` (too thick)
- ❌ Not `border-white` (wrong color)

**Icon:**
- ✅ `UserIcon size="md" variant="primary"` - AT&T Functional Blue
- ❌ Not white text
- ❌ Not muted gray

**Size:**
- ✅ `h-10 w-10` (40px) - Standard size
- ✅ `h-8 w-8` (32px) - Smaller contexts (Activity log)

## 🎨 **Two Standard Patterns**

### **Pattern 1: User Avatar with Icon** (Most Common)
```tsx
<div className="h-10 w-10 rounded-full bg-fw-blue-light flex items-center justify-center border border-fw-secondary">
  <UserIcon size="md" variant="primary" />
</div>
```

**Use Cases:**
- User lists
- Member lists
- Activity logs (without avatar image)
- Access control lists

### **Pattern 2: User Avatar with Initials** (When showing actual users)
```tsx
<div className="h-10 w-10 rounded-full bg-fw-link flex items-center justify-center border border-fw-secondary text-white font-medium text-sm">
  {user.name.charAt(0).toUpperCase()}
</div>
```

**Use Cases:**
- Group member avatars
- Pool user lists
- Any place showing actual user initials

**Note:** When initials are shown, use `bg-fw-link` (solid blue) for contrast with white text

---

## 🎨 **Flywheel Compliance**

### ✅ **Approved Colors:**
- `bg-fw-blue-light` - Light blue wash (for icon containers)
- `bg-fw-link` - Functional Blue (for initial containers)
- `bg-fw-wash` - Very light gray (for "+5 more" badges)
- `border-fw-secondary` - Standard border color
- `text-fw-body` - Body text color

### ❌ **Removed:**
- All gradients (`from-X to-Y`)
- Tailwind direct colors (`bg-blue-500`, `bg-gray-200`)
- Non-Flywheel borders (`border-white`, `border-2`)
- Inconsistent icon colors
